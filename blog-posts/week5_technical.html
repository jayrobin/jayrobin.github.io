Or: how I learned to stop worrying and love programming

Week Five - Technical blog

02 Jul 2014

What is Object Oriented Programming?

Object Oriented Programming, or OOP for short, is a programming paradigm which focuses on the creation of 'objects' which represent components a system (they could be representative of actual objects, or only make sense within the bounds of the system). Compare this to Procedural Programming, which is a programming paradigm centered around the logical sequence of step-by-step actions.

What are classes?

In OOP, classes are the blueprints for objects - they are defined by the programmer and created as 'instances' during runtime. Classes can contain attributes (also known as variables or properties) to store data about the object or class, and methods (also known as functions) to perform actions on the object or class.

The following is a class in ruby - it does not currently do anything or store any data about an individual instance:

class Vehicle end

However, we can still create new vehicles:

vehicle_1 = Vehicle.new vehicle_2 = Vehicle.new vehicle_3 = Vehicle.new

What are instance methods?

Instance methods are methods which are atteched to every instance of a class:

class SomeClass   def print_greeting     puts "Hello, world."   end end

To call the method above, we would create a new instance of SomeClass and call print_greeting on the instance:

some_object = SomeClass.new some_object.print_greeting # Hello, world.

What are instance variables?

Instance variables are variables attached to individual instances of a class. In our example above, we have three vehicles, but they do not store any information. We can change this by adding instance variables to the class, which are represented in ruby with @variable_name. One way we could set the variable is by passing a value into the 'initialize' method, which is called when a new instance is created.

class Vehicle   def initialize(car_model)     @model = car_model   end end

We can now set the model of each car, and retrieve it again later:

vehicle = Vehicle.new("Pontiac Aztek") puts vehicle.model # undefined method 'model'

Why couldn't we get our model variable from the vehicle in the same way that we called the print_greeting method? The reason is that instance variables in ruby are private, so they are only accessible from within the object itself. We can get around this by creating an accessor method:

class Vehicle   def initialize(car_model)     @model = car_model   end   def model     @model   end end

Now, when we try to set and retrieve the model for a car, it will work:

vehicle = Vehicle.new("Pontiac Aztek") puts vehicle.model # Pontiac Aztek

But what if we want to change this variable again later?

vehicle = Vehicle.new("Pontiac Aztek") vehicle.model = "Fiat Multipla" # undefined method 'model='

Unfortunately, we don't have a method in the Vehicle class to facilitate changing the model variable, but we can make one like so:

class Vehicle   def initialize(car_model)     @model = car_model   end   def model     @model   end   def model=(car_model)     @model = car_model   end end

And now when we try to change the model variable:

vehicle = Vehicle.new("Pontiac Aztek") vehicle.model = "Fiat Multipla" puts vehicle.model # Fiat Multipla

What else can we do with objects?

To build upon our vehicles example, we could create a Garage class to store vehicles, like a database. For this, we'd need the Garage class to have a way to store the individual vehicles - an array seems like a good choice:

class Garage   def initialize     @vehicles = []   end end

At this point, we could create a Garage object and it would have an empty list of vehicles, but we have no way to add vehicles to the list, and no way to get them afterwards. We could create an accessor method for the vehicles array:

class Garage   def initialize     @vehicles = []   end   def vehicles     @vehicles   end end

This would allow us to push vehicles onto the vehicles array:

vehicle = Vehicle.new("Kia Optima") garage = Garage.new garage.vehicles.push(vehicle)

However, opening up the internals of a class like this is not very good practice - after creating a garage we could easily (accidently or maliciously) alter the vehicles array in destructive ways. Instead, we'd be better creating an instance method to add a vehicle with the implementation of the method hidden within the class:

class Garage   def initialize     @vehicles = []   end   def add(vehicle)     @vehicles.push(vehicle)   end end vehicle = Vehicle.new("Kia Optima") garage = Garage.new garage.add(vehicle)

We could extend our garage further by adding some useful instance methods:

class Garage   def initialize     @vehicles = []   end   def add(vehicle)     @vehicles.push(vehicle)   end   def get_number_of_vehicles     @vehicles.size   end   def print_vehicles     @vehicles.each { |v| puts v }   end end

The get_number_of_vehicles method returns the number of vehicles in the garage, and the print_vehicles method will print out every vehicle in the garage.

While this article only covers the absolute basics of Object Oriented Programming, armed with just classes, instance variables and intance methods you can create some very complex systems in ruby.
