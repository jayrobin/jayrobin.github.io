Or: how I learned to stop worrying and love programming

Week Two - Technical blog

13 Jun 2014

The box model

On a web page, all of the various elements can be thought of as boxes containing the HTML tag 'content': a 'P' tag containing text, or an 'IMG' tag containing an image, for example. The CSS box model breaks this box down further into the content box itself, the padding around the content, the border around the padding, and the margin around the border, as shown in the image below.

Content

The content of an element is, in html, everything between the tag opening and the tag closing. In the case of a self-closing tag such as 'IMG', the content is the image referred to in the 'src' attribute.

The width and height of the content can be changed for 'block' elements (elements such as 'DIV' or that have been set as block using 'display: block;'):

img {   width: 100px;   height: 100px; }

Padding

The padding of an element is the space between the content and the border. It can be set in two ways, by changing the top/left/right/bottom padding individually:

div {   padding-top: 10px;   padding-bottom: 20px; }

Or by using shorthand to set multiple sides at once:

div {   padding: 10px; }

The above code will set the padding on every side to 10px, but we can set sides indivually using the same shorthand:

div {   padding: 10px 20px 30px 40px; }

The order is important and can be remembered as a clock-face: top, right, bottom, left. If two or three sides are defined in shorthand, the value for missing side is determined by the opposing side:

div {   /* top, left and right, bottom */   padding: 10px 20px 30px; } div {   /* top and bottom, left and right */   padding: 10px 20px; }

Border

Though borders around elements are hidden by default, they can be shown and styled as follows:

p {   /* sets border width, style and color */   border: 1px solid black; }

The top, left, bottom and right borders can also be set individually:

p {   border-bottom: 2px dashed #aaa; }

Margin

The margins are what space the elements out on a page and are defined in a very similar way to the padding of an element, but create space outside the border.

div {   margin-top: 10px;   margin-right: 10px;   margin-bottom: 20px;   margin-left: 10px; }

Or, in shorthand:

div {   margin: 10px 10px 20px 10px; }

Or, in even shorter hand...

div {   margin: 10px 10px 20px; }

Total width/height calculation

One thing to bear in mind when sizing elements is that setting the 'width' or 'height' of an element only sets the size of the content. The total width, for example, is defined as content width + left padding + right padding + left border + right border + left margin + right margin. This can cause confusion when attempting to accurately size and place elements on a page, but thankfully we can make it a little easier for ourselves...

Box-sizing

The 'box-sizing' attribute in CSS determines how the 'width' and 'height' elements should be used when calculating the total width and height. By default it is set to 'content-box' which means that the 'width' and 'height' attributes only refer to the content width and height, but by setting it to 'border-box', it will take them as the content+padding+borders.

For example, say we wanted a div that was a total of 100px wide not including the margins, with 1px borders and 10px padding, we would have to do something like this:

div {   border: 1px solid black;   padding: 10px;   width: 78px;   /* 100px - borders - padding */   /* =100px - (2*1px) - (2*10px) */   /* =100px - 2px - 20px */   /* =78px */ }

However, if we instead set the box-sizing to 'border-box', we don't need to do that manual calculation and can leave the width as 100px and still get the same total width:

div {   box-sizing: border-box;   border: 1px solid black;   padding: 10px;   width: 100px; }
